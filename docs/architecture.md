# Архитектура проекта

## Задание 1: Начальная структура

На данном этапе проект представляет собой скелетную структуру для будущих модулей.

### Модули:
- **`main.cpp`**: Точка входа в приложение, пока выводит приветствие.
- **`obd_parser.h` / `obd_parser.cpp`**: Будет отвечать за чтение данных телеметрии.
- **`onnx_classifier.h` / `onnx_classifier.cpp`**: Будет использоваться для классификации стиля вождения.
- **`dashboard.h` / `dashboard.cpp`**: Модуль для отрисовки приборной панели.
- **`dms_monitor.h` / `dms_monitor.cpp`**: Модуль системы мониторинга водителя (DMS).
- **`dms_hud.h` / `dms_hud.cpp`**: Модуль для отображения информации DMS на HUD.

### Система сборки:
Используется CMake для управления сборкой проекта. Он настроен на:
- Стандарт C++17.
- Поиск библиотек OpenCV и ONNX Runtime.
- Подключение Google Test для модульного тестирования.
- Генерацию двух исполняемых файлов: `RealCarMonitor` (основное приложение) и `RunTests` (для запуска тестов).

### Контроль версий:
Используется Git для контроля версий и GitHub для удаленного репозитория. Файл `.gitignore` исключает временные и сборочные файлы.
